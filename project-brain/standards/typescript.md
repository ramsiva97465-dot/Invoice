# TypeScript Standards

- Use strict mode (`"strict": true` in tsconfig).
- Prefer `interface` for object shapes.
- Use `readonly` for immutable fields.
- Enforce consistent naming: `camelCase` for variables/functions, `PascalCase` for classes/types.
- Include JSDoc comments for exported symbols.
- Enable `noImplicitAny` and `noUnusedLocals`.
- Use `eslint` with `@typescript-eslint` recommended rules.
